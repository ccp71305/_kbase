# AWS SDK Upgrade Plan for WebBL Module

## Overview

This document outlines the comprehensive plan to upgrade the WebBL (Web Bill of Lading) module from AWS SDK 1.x to AWS SDK 2.x using the cloud-sdk abstraction libraries (`cloud-sdk-api` and `cloud-sdk-aws`). This upgrade follows the same patterns established in the booking-bridge module migration.

## Migration Phases

### Phase 1: SQS Migration (Low Risk)

**Duration:** 1-2 days

**Scope:**
- Replace `AmazonSQS` with `MessagingClient<String>` from cloud-sdk
- Update `SQSClient.java` to use cloud-sdk API
- Update `SQSListener.java` to use `QueueMessage<String>`
- Create `WebBLMessagingModule.java`

### Phase 2: SNS Migration (Low Risk)

**Duration:** 1-2 days

**Scope:**
- Replace `AmazonSNS` with `NotificationService` from cloud-sdk
- Update `SNSClient.java` to use cloud-sdk API
- Add SNS provider to `WebBLMessagingModule.java`

### Phase 3: S3 Migration (Low Risk)

**Duration:** 1-2 days

**Scope:**
- Replace `AmazonS3` with `StorageClient` from cloud-sdk
- Update `S3WorkspaceService.java` to use cloud-sdk API
- Create `WebBLStorageModule.java`

### Phase 4: DynamoDB Migration (High Risk)

**Duration:** 3-4 days

**Scope:**
- Replace DynamoDB annotations (SDK 1.x → SDK 2.x)
- Migrate `BLVersion.java` model
- Remove custom `DateToEpochSecond` converter (use cloud-sdk's `DateEpochSecondAttributeConverter`)
- Update `BLDao` to use `DatabaseRepository`
- Create `WebBLDynamoModule.java`
- Preserve GSI definition (`INTTRA_REFERENCE_NUMBER_INDEX`)

### Phase 5: Test Migration

**Duration:** 2-3 days

**Scope:**
- Update unit tests to mock cloud-sdk interfaces
- Create DynamoDB integration tests using `BaseDynamoDbIT` pattern
- Update existing integration test framework

### Phase 6: Integration & Verification

**Duration:** 2-3 days

**Scope:**
- End-to-end testing
- Performance validation
- Rollback testing

#### Step 1.1: Create WebBLMessagingModule

**New File: `WebBLMessagingModule.java`**
```java
package com.inttra.mercury.webbl.config;

import com.google.inject.AbstractModule;
import com.google.inject.Provides;
import com.google.inject.Singleton;
import com.inttra.mercury.cloudsdk.messaging.api.MessagingClient;
import com.inttra.mercury.cloudsdk.messaging.factory.MessagingClientFactory;
import com.inttra.mercury.cloudsdk.notification.api.NotificationService;
import com.inttra.mercury.cloudsdk.notification.factory.NotificationClientFactory;
import lombok.extern.slf4j.Slf4j;

/**
 * Guice module for cloud-sdk messaging and notification configuration for WebBL service.
 * Provides MessagingClient (SQS) and NotificationService (SNS) using cloud-sdk-aws factories.
 * 
 * <p>This module follows the same pattern as BookingBridgeMessagingModule,
 * providing modular configuration for AWS messaging services.</p>
 */
@Slf4j
public class WebBLMessagingModule extends AbstractModule {
    
    private final WebBLConfig config;
    
    public WebBLMessagingModule(WebBLConfig config) {
        if (config == null) {
            throw new IllegalArgumentException("config must not be null");
        }
        this.config = config;
    }
    
    @Override
    protected void configure() {
        log.info("Configuring WebBL Messaging module (SQS/SNS) ...");
        bind(WebBLConfig.class).toInstance(config);
    }
    
    /**
     * Provides MessagingClient for SQS using cloud-sdk-aws factory.
     * Uses default AWS configuration (region from environment, default credentials).
     */
    @Provides
    @Singleton
    public MessagingClient<String> provideMessagingClient() {
        log.info("Creating SQS MessagingClient using cloud-sdk-aws factory");
        MessagingClient<String> messagingClient = MessagingClientFactory.createDefaultStringClient();
        log.info("SQS MessagingClient created successfully");
        return messagingClient;
    }
    
    /**
     * Provides NotificationService for SNS using cloud-sdk-aws factory.
     */
    @Provides
    @Singleton
    public NotificationService provideNotificationService() {
        log.info("Creating SNS NotificationService using cloud-sdk-aws factory");
        
        String topicArn = config.getSnsEventTopicArn();
        if (topicArn == null || topicArn.trim().isEmpty()) {
            throw new IllegalStateException("WebBL SNS event topic ARN is not configured");
        }
        
        NotificationService notificationService = NotificationClientFactory.createDefaultClient(topicArn);
        log.info("SNS NotificationService created successfully for topic: {}", topicArn);
        
        return notificationService;
    }
}
```

```java
package com.inttra.mercury.webbl.common.messaging;

import com.google.inject.Inject;
import com.google.inject.Singleton;
import com.inttra.mercury.cloudsdk.messaging.api.MessagingClient;
import com.inttra.mercury.cloudsdk.messaging.api.QueueMessage;
import com.inttra.mercury.cloudsdk.messaging.model.ReceiveMessageOptions;
import lombok.extern.slf4j.Slf4j;

import java.util.List;

/**
 * SQS Client wrapper for sending, receiving, and deleting messages from SQS queues.
 * Uses cloud-sdk-api MessagingClient abstraction for vendor-neutral messaging.
 * Migrated from AWS SDK 1.x to cloud-sdk-aws (AWS SDK 2.x wrapper).
 */
@Slf4j
@Singleton
public class SQSClient {
    private static final String FAILED_ATTEMPTS = "failedAttempts";
    private final MessagingClient<String> messagingClient;

    @Inject
    public SQSClient(MessagingClient<String> messagingClient) {
        this.messagingClient = messagingClient;
    }

    /**
     * Sends a message to the specified SQS queue.
     *
     * @param target  The queue URL
     * @param content The message content
     */
    public void sendMessage(String target, String content) {
        try {
            messagingClient.sendMessage(target, content);
            log.debug("SQS message sent successfully to queue: {}", target);
        } catch (Exception e) {
            log.error("Exception while sending to queue: {}", target, e);
            throw new RuntimeException(String.format("Sending failed to %s: %s", target, e.getMessage()), e);
        }
    }

    /**
     * Deletes a message from the specified SQS queue.
     *
     * @param queueUrl             The queue URL
     * @param messageReceiptHandle The message receipt handle
     */
    public void deleteMessage(String queueUrl, String messageReceiptHandle) {
        try {
            messagingClient.deleteMessage(queueUrl, messageReceiptHandle);
            log.debug("SQS message deleted successfully from queue: {}", queueUrl);
        } catch (Exception e) {
            log.error("Exception while deleting message from queue: {}", queueUrl, e);
            throw e;
        }
    }

    /**
     * Receives messages from the specified SQS queue.
     *
     * @param queueUrl        The queue URL
     * @param maxMessages     Maximum number of messages to receive
     * @param waitTimeSeconds Wait time for long polling
     * @return List of received messages
     */
    public List<QueueMessage<String>> receiveMessage(String queueUrl, int maxMessages, int waitTimeSeconds) {
        try {
            ReceiveMessageOptions options = ReceiveMessageOptions.builder()
                    .queueUrl(queueUrl)
                    .maxMessages(maxMessages)
                    .waitTimeSeconds(waitTimeSeconds)
                    .build();

            List<QueueMessage<String>> messages = messagingClient.receiveMessages(options);
            log.debug("Received {} messages from queue: {}", messages.size(), queueUrl);
            return messages;
        } catch (Exception e) {
            log.error("Exception while receiving messages from queue: {}", queueUrl, e);
            throw e;
        }
    }
}
```

```java
import java.io.File;
import java.nio.charset.StandardCharsets;

/**
 * S3 Workspace Service for file operations.
 * Uses cloud-sdk-api StorageClient abstraction for vendor-neutral storage.
 * Migrated from AWS SDK 1.x to cloud-sdk-aws (AWS SDK 2.x wrapper).
 */
@Slf4j
@Singleton
public class S3WorkspaceService implements WorkspaceService {
    private final StorageClient storageClient;

    @Inject
    public S3WorkspaceService(StorageClient storageClient) {
        this.storageClient = storageClient;
    }

    @Override
    public void copyObject(String sourceBucket, String sourceFileName,
                           String targetBucket, String targetFileName) {
        try {
            boolean success = storageClient.copyObject(sourceBucket, sourceFileName, targetBucket, targetFileName);
            if (!success) {
                throw new RuntimeException(String.format("Failed to copy object: %s/%s -> %s/%s",
                        sourceBucket, sourceFileName, targetBucket, targetFileName));
            }
            log.debug("S3 object copied successfully: {}/{} -> {}/{}", 
                    sourceBucket, sourceFileName, targetBucket, targetFileName);
        } catch (Exception e) {
            log.error("Error copying S3 object: {}/{} -> {}/{}", 
                    sourceBucket, sourceFileName, targetBucket, targetFileName, e);
            throw e;
        }
    }

    @Override
    public void putObject(String bucket, String fileName, String content) {
        try {
            storageClient.putObject(bucket, fileName, content);
            log.debug("S3 object put successfully: {}/{}", bucket, fileName);
        } catch (Exception e) {
            log.error("Error putting S3 object: {}/{}", bucket, fileName, e);
            throw e;
        }
    }

    @Override
    public String getContent(String bucket, String fileName) {
```

4. **Revert Model Classes**: Restore SDK 1.x DynamoDB annotations
5. **Restore DAO Classes**: Restore DynamoDBCrudRepository extension
6. **Revert Service Classes**: Restore SDK 1.x client usage

#### Compatibility Summary

The legacy messaging-helper classes and their cloud-sdk-api replacements are **API-identical**.
Only the package namespace differs — all fields, Builder patterns, method signatures, constants,
and functional interfaces are the same. Code consuming these classes required **zero logic changes**;
only `import` statements were updated.

| Aspect | Legacy (messaging-helper) | Cloud-SDK-API |

| `MetaData` | Same fields: `messageId`, `workflowId`, `rootWorkflowId`, `parentWorkflowId`, `bucket`, `fileName`, `component`, `exitStatus`, `timestamp`, `projections`. Same `Builder`, `parseJson()`, `toJsonString()` methods. Same `Projection` constants. | Identical API. Internal JSON handled by `JsonSupport` instead of `Json`. |
| `Event` | Same fields, `Builder`, `SubType` enum (`START_WORKFLOW`, `CLOSE_WORKFLOW`), `Token`/`MessageToken` constants. | Identical API. Internal timestamps via `DateConstants` instead of `Json`. |
| `EventLogger` | Same 3 overloads of `logCloseRunEvent()`. Uses `javax.inject.Inject`. | Identical API. Uses `com.google.inject.Inject` (transparent to Guice). |
| `EventGenerator` | Same constants (`CLOSE_RUN`, `DROP_OFF_QUEUE`, `PICK_UP_QUEUE`), same `getCloseRunEvent()` signature. | Identical API. |

**Key difference in DI annotations:** Legacy uses `javax.inject.*`, cloud-sdk uses `com.google.inject.*`.
This is transparent to Guice since it supports both annotation types.
